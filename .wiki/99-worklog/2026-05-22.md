# 2026-05-22 작업 로그

## 요약

- 사용자 지시에 따라 `.wiki`를 다단 구조의 장기 지식저장소로 만들고, 이번 범위는 끝까지 위키 개정으로 제한했다.
- Claude Code snapshot, Agentica 현재 저장소, 외부 reference를 읽고 Agentica 강화 설계로 번역했다.
- `MicroAgentica`는 변경 대상에서 제외한다는 원칙을 모든 설계 문서에 반복 반영했다.
- 최종적으로 세 축 대조 리뷰를 `00-governance/total-review.md`에 추가했고, 긴 진행 로그는 핵심 이력만 남기도록 정리했다.

## 운영 원칙 확정

- 장기 지식은 주제 문서에 둔다.
- 날짜별 로그는 진행 감사 이력만 남긴다.
- Claude Code 코드는 복사하지 않고 아키텍처 패턴만 참조한다.
- `@agentica/vector-selector`의 DBMS/embedding 의존 구조는 core 기본 경로로 가져오지 않는다.
- 구현 PR은 이 위키를 근거로 phase별로 작게 나눈다.

## Agentica 현재 구조 독해

- 루트 workspace, package scripts, README, package inventory를 `01-project`와 `04-agentica/package-inventory.md`에 정리했다.
- `packages/core`의 `Agentica`, `MicroAgentica`, `orchestrate/*`, context, history/event/json, functional/controller, request/streaming utils를 문서화했다.
- `packages/vector-selector`는 sqlite/postgres/embedding 의존 구조와 core 기본 경로로 부적합한 이유를 정리했다.
- `packages/rpc`, `packages/chat`은 public surface와 renderer 영향면을 따로 문서화했다.
- `packages/benchmark`, `test`, `benchmark/vector-selector-benchmark`, CLI/examples/internal fixtures를 검증 전략 근거로 정리했다.
- `packages/cli`, `packages/create-agentica`, `website`, `articles`, `docs`의 문서/스캐폴드 영향면을 정리했다.

## Claude Code 핵심 구조 독해

- `query.ts`와 `query/*`를 읽고 stateful loop, context prep, compact/recovery, streaming tool execution, model fallback, attachment drain, stop hook, token budget continuation을 정리했다.
- tool contract와 execution path, permission flow, tool hooks, result projection, synthetic result 원칙을 분석했다.
- `ToolSearchTool`과 `utils/toolSearch.ts`를 읽고 DB 없는 lexical search, direct `select:`, deferred schema loading, discovered tool state를 Agentica local selector 설계로 번역했다.
- compact service를 읽고 full/partial/micro/session memory compact와 preserved boundary, post-compact cleanup, prompt-too-long recovery를 정리했다.
- memory, skill, plugin, MCP loading 구조를 읽고 lazy capability loading과 registry/reload boundary를 설계 문서에 반영했다.

## Claude Code 도구와 정책 독해

- FileRead/Grep/FileEdit/FileWrite/Glob/Notebook/Todo/TaskOutput/LSP/TaskStop 계열을 읽고 operation policy/state 설계에 반영했다.
- BashTool 실행, sandbox, parser trust gate, legacy security validator, PowerShell/REPL/platform shell gate를 분석했다.
- permission rule, PermissionUpdate, managed policy, PreToolUse hook과 UI permission prompt를 control flow 문서로 정리했다.
- slash command runtime과 catalog를 읽고 command result union, headless/remote/bridge allowlist, registry/local machine mutation을 분리했다.

## Claude Code runtime과 adapter 독해

- MCP service, auth, SDK/in-process transport, `/mcp` UI/CLI, approval flow를 connector runtime 관점으로 정리했다.
- plugin marketplace/install/update/autoupdate/validate/MCPB/cache/reload/startup/headless 흐름을 capability registry 설계로 번역했다.
- task/coordinator runtime, disk output, progress, LocalShell/LocalAgent/RemoteAgent/MainSession/InProcessTeammate/DreamTask를 task 설계 근거로 정리했다.
- remote/bridge/server/CLI transport/SDK control schema를 읽고 remote session state, ack/sequence/worker epoch, public surface 비노출 원칙을 정리했다.
- state/swarm runtime, mailbox, tmux/iTerm2 pane backend, teammate context를 읽고 team/member runtime을 optional adapter 후보로 정리했다.
- Ink/UI renderer를 읽고 model context projection과 render projection 분리, large result preview/reference, selection/search/scroll/input state 비노출 원칙을 정리했다.
