# 2026-05-24 작업 로그

## 요약

- 최초 `.wiki` 전수 리뷰에서는 첫 구현 대상으로 local operation index를 유지했고, 후속 재리뷰에서는 selector 이후의 context/result/progressive-read 최적화를 다음 우선순위로 재정렬했다.
- 외부 reference를 다시 확인해 OpenAI가 `tool_search`를 공식 문서에 추가했음을 반영했다. 다만 GPT-5.4+ 등 provider/model 제약이 있어 Agentica core 기본 경로는 여전히 provider-neutral local selector가 맞다.
- AutoBe public source의 in-memory/hybrid RAG 구현을 읽고, section granularity, BM25 baseline, deterministic index cache, dynamic K, schema projection 방식을 Agentica 적용 관점으로 정리했다.
- `packages/core`에 dependency-free `AgenticaOperationIndex`와 opt-in `IAgenticaConfig.selector`를 구현했다.

## 구현

- `AgenticaOperationIndex`
  - operation/controller/function/description/parameter/schema/http/protocol/searchHint field 색인
  - camel/snake/kebab/path/Unicode tokenizer
  - field-weighted lexical scoring
  - `select:<operationKey>` direct selection
  - `+required` term pre-filter
  - deterministic registry version

- `orchestrate.select`
  - 기본값은 기존 LLM selector 유지
  - `local`은 local top-K를 바로 stack에 push
  - `hybrid`는 local top-K 후 기존 LLM selector로 재선택
  - `auto`는 schema character heuristic으로 기존 LLM selector와 hybrid를 전환
  - local no-match 기본 fallback은 기존 LLM selector
  - LLM selector validation은 “전체 operation 존재 여부”가 아니라 “이번 candidate catalog에 포함되는지”로 보정

- `MicroAgentica`
  - 변경 없음

## 검증

```bash
pnpm --filter @agentica/core exec vitest run src/selector/AgenticaOperationIndex.spec.ts
pnpm --filter @agentica/core exec tsc --noEmit
pnpm --filter @agentica/core test -- --run
pnpm --filter @agentica/core lint
```

위 명령은 통과했다. `lint`는 기존 `functional/*` JSDoc warning 20개를 출력하지만 exit code는 0이다.

## 후속 재리뷰: Claude Code 최적화 범위 확장

사용자 피드백에 따라 selector 중심 결론을 재검토했다. Local selector는 첫 구현으로 맞았지만, Agentica가 general purpose agent로 Claude Code에서 더 크게 배워야 할 부분은 다음 축이다.

- context projector와 result budget
- 큰 result의 preview/reference materialization
- file splitting을 일반화한 progressive read/segmented content protocol
- digest/version 기반 unchanged stub
- read-state/freshness precondition
- compact lifecycle과 restore attachment
- transition-aware runtime loop
- task output cursor, stop, notification queue
- model-facing context와 chat render projection 분리
- raw history가 아니라 projected request 기준의 context diagnostics
