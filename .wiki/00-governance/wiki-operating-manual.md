# 위키 운영 매뉴얼

## 목적

이 `.wiki`는 Agentica 강화 작업을 위한 장기 지식저장소다. 목표는 새 세션이 위키만 읽어도 다음을 파악할 수 있게 하는 것이다.

- Claude Code에서 어떤 구조를 배울지
- Agentica의 현재 구조가 어디서 약한지
- `MicroAgentica`를 건드리지 않는 이유와 범위
- 어떤 파일을 읽었고, 어떤 파일은 아직 미독해인지
- 이후 구현 시 어떤 순서와 검증 기준을 따라야 하는지

## 쓰기 원칙

- 확인한 사실은 파일 경로와 가능하면 line anchor를 함께 기록한다.
- 추론은 "결론", "판단", "적용 결정" 같은 제목 아래 사실과 분리한다.
- 전수정독이 끝나지 않은 영역은 완료처럼 쓰지 않는다.
- 작업 로그는 날짜별로 남기고, 장기 지식은 주제 문서로 옮긴다.
- 코드 변경이 없는 분석 턴도 반드시 `reading-ledger.md` 또는 관련 문서에 흔적을 남긴다.

## 문서 계층

- `00-governance`: 위키 운영, 완료 조건, 독해 상태
- `01-project`: Agentica 저장소 자체의 전체 지도
- `02-research`: Claude Code 및 외부 reference 분석
- `03-design`: Agentica Next 설계와 구현 계획
- `04-agentica`: 본 저장소 패키지별 소스 구조와 영향면
- `99-worklog`: 날짜별 진행 로그

## 다음 세션 시작 절차

1. `00-index.md`를 연다.
2. `00-governance/reading-ledger.md`에서 완료/미완료 영역을 확인한다.
3. 현재 작업이 설계인지 구현인지 확인한다.
4. 구현 전이면 `03-design`과 `04-agentica` 문서를 먼저 갱신한다.
5. 구현 후에는 변경된 코드와 관련된 위키 문서를 반드시 갱신한다.

## 금지 사항

- `MicroAgentica` 변경을 전제로 설계하지 않는다.
- Claude Code 코드를 복사하지 않는다. 공개 snapshot의 아키텍처 패턴만 참조한다.
- `@agentica/vector-selector`의 DBMS 의존 구조를 core 기본 경로로 끌고 오지 않는다.
- 전수정독이 끝나지 않았는데 "완료"라고 표시하지 않는다.
