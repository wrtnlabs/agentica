```typescript filename="src/main.ts" showLineNumbers
import { Agentica } from "@agentica/core";
import OpenAI from "openai";

const agent = new Agentica({
  vendor: {
    api: new OpenAI({ apiKey: process.env.OPENAI_API_KEY }),
    model: process.env.OPENAI_MODEL ?? "gpt-4o-mini",
  },
  controllers: [...],
});

agent.on("userMessage", async (event) => {
  console.log("user", event.contents);
});

agent.on("assistantMessage", async (event) => {
  for await (const text of event.stream)
    process.stdout.write(text);
  process.stdout.write("\n");
});

agent.on("initialize", async () => {
  console.log("Function calling started");
});

agent.on("select", async (event) => {
  console.log(
    "Candidate function selected",
    event.selection.operation.name,
    event.selection.reason,
  );
});

agent.on("execute", async (event) => {
  console.log(
    "Function executed",
    event.operation.name,
    event.arguments,
    event.value,
  );
});

agent.on("describe", async (event) => {
  for (const execute of event.executes)
    console.log(`  - ${execute.operation.name}`);
  console.log(await event.join());
});

await agent.conversate("I wanna buy Surface Pro");
```

`Agentica.conversate()` returns the histories created during the turn. Event listeners let you stream the same process to a UI as it happens.

The current message events are `userMessage` and `assistantMessage`. Older `text` event examples are obsolete.

`userMessage` contains the user contents that entered the turn. `assistantMessage` streams ordinary assistant text and can be consumed either with `for await` over `event.stream` or by awaiting `event.join()`.

`describe` is emitted after function executions when Agentica asks the model to explain the results. Descriptions can be large when API responses are large, so streaming is usually better for UI rendering.




## Functional Events
<Tabs items={[
  <code>src/main.ts</code>,
  <code>AgenticaEvent</code>,
]}>
  <Tabs.Tab>
```typescript filename="src/main.ts" showLineNumbers
import { Agentica } from "@agentica/core";
import OpenAI from "openai";

const agent = new Agentica({
  vendor: {
    api: new OpenAI({ apiKey: process.env.OPENAI_API_KEY }),
    model: process.env.OPENAI_MODEL ?? "gpt-4o-mini",
  },
  controllers: [...],
});

agent.on("initialize", () => {
  console.log("function calling initialized");
});

agent.on("select", (event) => {
  console.log(event.selection.operation.name, event.selection.reason);
});

agent.on("call", (event) => {
  console.log(event.operation.name, event.arguments);
});

agent.on("cancel", (event) => {
  console.log("cancelled", event.selection.operation.name);
});

agent.on("execute", (event) => {
  console.log(event.operation.name, event.success, event.value);
});

agent.on("validate", (event) => {
  console.log(event.operation.name, event.result.errors);
});

agent.on("jsonParseError", (event) => {
  console.log(event.operation.name, event.failure);
});

await agent.conversate("I wanna buy Surface Pro");
```
  </Tabs.Tab>
  <Tabs.Tab>
    <AgenticaEventSnippet />
  </Tabs.Tab>
</Tabs>

These events expose the function calling pipeline:

- `initialize`: function calling mode starts.
- `select`: an operation is selected as a candidate.
- `call`: the model composed arguments before execution.
- `cancel`: a selected operation was cancelled.
- `execute`: an operation finished, successfully or not.
- `validate`: argument validation failed and will be fed back to the model.
- `jsonParseError`: the model returned malformed JSON arguments.

`call.arguments` is mutable. Changing it changes the actual function request, so only mutate it when your application owns that correction path.




## Request and Response Events
<Tabs items={[
  <code>src/main.ts</code>,
  <code>AgenticaEvent</code>,
]}>
  <Tabs.Tab>
```typescript filename="src/main.ts" showLineNumbers {11-20}
import { Agentica } from "@agentica/core";
import OpenAI from "openai";

const agent = new Agentica({
  vendor: {
    api: new OpenAI({ apiKey: process.env.OPENAI_API_KEY }),
    model: process.env.OPENAI_MODEL ?? "gpt-4o-mini",
  },
  controllers: [...],
});

agent.on("request", (event) => {
  console.log(event.source, event.stream, event.body);
});

agent.on("response", async (event) => {
  console.log(event.source, event.stream, event.request_id);
  const completion = await event.join();
  console.log(completion.usage);
});

await agent.conversate("I wanna buy Surface Pro");
```

`request` and `response` are low-level OpenAI-compatible API tracing events. Use them for logging, instrumentation, and controlled request mutation.
