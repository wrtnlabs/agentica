## Current Focus

Agentica's current direction is to make function calling reliable at large controller scale without forcing users to design custom agent graphs.

Recently shipped work:

- `config.selector` in `@agentica/core` for built-in `llm`, `local`, `hybrid`, and `auto` operation selection.
- `AgenticaOperationIndex`, a dependency-free local operation index used by the built-in selector.
- Updated event and history APIs centered on `userMessage`, `assistantMessage`, `select`, `call`, `execute`, and `describe`.
- Optional `@agentica/vector-selector` for semantic retrieval with SQLite, PostgreSQL, or a custom embedding strategy.




## Large Tool Catalogs

Large catalogs remain the most important Agentica performance problem. The goal is to reduce token pressure while keeping tool selection explainable and observable.

Near-term work:

- benchmark `llm`, `local`, `hybrid`, `auto`, and vector-selector strategies on the same scenarios,
- expose better diagnostics for why a selector chose or rejected operations,
- document operation metadata conventions such as `searchHint` and `loadPolicy`,
- continue improving benchmark reports for selection quality and execution quality.




## Local LLMs

Agentica still plans to support local LLMs that do not have native function calling. The intended direction is a prompt-template adapter plus the same validation feedback strategy used by OpenAI-compatible function calling.

The built-in selector helps this roadmap because it can shrink the function catalog before a weaker or smaller model sees the caller prompt.




## Benchmarks

`@agentica/benchmark` currently focuses on single-turn selection and call quality. Multi-turn benchmarking remains planned because real agents need to preserve context, recover from partial information, and validate execution behavior across several user turns.

The benchmark roadmap should compare:

- selector quality,
- argument validation recovery,
- execution success rate,
- token usage,
- latency,
- multi-turn context retention.




## Hosted Retrieval

OpenAI-hosted tool search and vector-store workflows are useful references, but Agentica does not currently depend on those hosted APIs for its core selector. The built-in selector runs locally, and `@agentica/vector-selector` uses pluggable SQLite/PostgreSQL/custom retrieval strategies.
