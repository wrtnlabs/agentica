## Vector Selector

`@agentica/vector-selector` is an optional selector plugin for large controller catalogs. It embeds operation descriptions, searches semantically relevant tools, then runs Agentica's selection prompt against that reduced candidate list.

Use the built-in [`config.selector`](/docs/core/selector) first when lexical local search is enough. Use this plugin when semantic retrieval and persistent embeddings are worth the extra infrastructure.




## Installation

```bash copy
npm install @agentica/vector-selector
```

SQLite mode also needs:

```bash copy
npm install better-sqlite3
```




## Usage

The vector selector replaces `executor.select`. This is lower-level than `config.selector` because the plugin owns the selection step.

```typescript filename="src/main.sqlite.ts" showLineNumbers {4-5, 13-23}
import { Agentica } from "@agentica/core";
import { BootAgenticaVectorSelector } from "@agentica/vector-selector";
import { configureSqliteStrategy } from "@agentica/vector-selector/strategy";
import Database from "better-sqlite3";
import OpenAI from "openai";

const selectorExecute = BootAgenticaVectorSelector({
  strategy: configureSqliteStrategy({
    db: new Database("agentica-vector-selector.sqlite"),
    cohereApiKey: process.env.COHERE_API_KEY!,
  }),
});

const agent = new Agentica({
  vendor: {
    api: new OpenAI({ apiKey: process.env.OPENAI_API_KEY }),
    model: process.env.OPENAI_MODEL ?? "gpt-4o-mini",
  },
  controllers: [...],
  config: {
    executor: {
      select: selectorExecute,
    },
  },
});

await agent.conversate("I wanna buy MacBook Pro");
```

import type { AgenticaContext } from "@agentica/core";
import type { IAgenticaVectorSelectorStrategy } from "@agentica/vector-selector";

import { Agentica } from "@agentica/core";
import { BootAgenticaVectorSelector } from "@agentica/vector-selector";

class CustomStrategy implements IAgenticaVectorSelectorStrategy {
  async searchTool(
    ctx: AgenticaContext,
    query: string,
  ): Promise<Array<{ name: string; description: string | undefined }>> {
    return searchYourVectorStore(ctx, query);
  }

  async embedContext(props: {
    ctx: AgenticaContext;
    setEmbedded: () => void;
  }): Promise<void> {
    await embedYourOperationCatalog(props.ctx);
    props.setEmbedded();
  }
}

const selectorExecute = BootAgenticaVectorSelector({
  strategy: new CustomStrategy(),
});

const agent = new Agentica({
  vendor: ...,
  controllers: [...],
  config: {
    executor: {
      select: selectorExecute,
    },
  },
});

## How It Works

`BootAgenticaVectorSelector()` returns an `IAgenticaExecutor.select` function. On the first matching context, it asks the strategy to embed the operation catalog. For each user turn it:

1. extracts search queries from the conversation,
2. calls `strategy.searchTool(ctx, query)` for each query,
3. removes duplicate tool names,
4. asks the selector prompt to choose functions from the retrieved tool list.

The plugin supports HTTP and class controllers through Agentica operations. HTTP operations include method, path, and tags in the reduced tool list when available.




## Strategies

### SQLite

`configureSqliteStrategy()` stores embeddings locally with `better-sqlite3` and `sqlite-vec`. It currently uses Cohere `embed-multilingual-light-v3.0`, so it requires `COHERE_API_KEY`.

Use SQLite for local development, small deployments, and benchmark runs where a single process owns the embedding database.

### PostgreSQL

`configurePostgresStrategy()` talks to Connector Hive and its retrieval API. The common local setup is:

```bash copy
docker run -d \
  --name postgres-vector \
  -e POSTGRES_USER=agentica \
  -e POSTGRES_PASSWORD=agentica \
  -e POSTGRES_DB=agentica \
  -p 5432:5432 \
  pgvector/pgvector
```

Then run Connector Hive with `DATABASE_URL`, `COHERE_API_KEY`, and optional API-key settings as required by that service.

Use PostgreSQL when you need a shared retrieval service or a deployment topology where multiple Agentica processes share the same operation embedding store.

### Custom

Implement `IAgenticaVectorSelectorStrategy` when you already have an embedding pipeline or want to use another vector database. The contract is intentionally small:

- `embedContext`: index the current operation catalog and call `setEmbedded()` when done.
- `searchTool`: return matching tool names and optional descriptions for a query.




## Built-In Selector vs Vector Selector

The built-in selector in `@agentica/core` is dependency-free and runs locally. It is usually the first choice because it needs no embedding model, database, or warm-up step.

The vector selector can outperform lexical search when operation names are weak, user wording differs from API wording, or you need semantic retrieval over a very large catalog. It costs extra setup and an embedding provider call path.

OpenAI also documents hosted tool search and vector-store-oriented workflows. Agentica's vector selector is not a wrapper around those hosted APIs today; it is an Agentica executor plugin with SQLite/PostgreSQL/custom strategy hooks. See:

- [OpenAI Tool Search](https://developers.openai.com/api/docs/guides/tools-tool-search)
- [OpenAI Function Calling](https://developers.openai.com/api/docs/guides/function-calling)
